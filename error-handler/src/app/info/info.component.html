Project Information

The purpose of this project is to demonstrate customizable error pages using Angular 9. The error handling is provided by an Error Component and an Error Service. Each try / catch or catch callback passes error information to the Error Service. The call back can then route to the appropriate error message page.

The Error Component is just a display component. It retrieves a custom error object from the Error Service. Angular has an interesting Directive called NgSwitch, which works in conjunction with *ngSwitchCase and *ngSwitchDefault. The NgSwitch directive is bound to a variable from the component. In this case, it's the status code. The *ngSwitchCase is applied to whatever HTML element that should be displayed based on the current value of the error status code.

Adding a new customized error page to your application is as simple as adding the appropriate *ngSwitchCase content to the Error Component and ensuring the use-cases for that error properly pass the correct information to the Error Service. This is a flexible, extensible error handling process that any can provide dynamic messaging content without much up-keep.
